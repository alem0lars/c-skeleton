    C Skeleton

    Author

    • Alessandro Molari

    Vision / Requirements

    I don't like Make or CMake so I have created a small (and portable between projects) build system using Rake, with the following main features:

    • Compile the code
    • Generate the reports (e.g. the documentation)
    • Run the software (in standalone mode) and the unit tests
    • Perform some kind of code analysis (static, dynamic, memory leaks, ...) The build system should be configurable but easy-to-use, so no mandatory configurations should be needed (a.k.a. 0-time setup)

    Usage

    rake <task> where task is:

    • compile: Compile the sources (mainly using cc)
    • run: Run the software (in standalone mode)
    • run_tests: Unit testing (using CUnit)
    • mem_leaks_analysis: Check for memory leaks (using valgrind)
    • static_analysis: Perform the static analysis (using splint)
    • doc: Generate the documentation (using doxygen)

    There are also other rake tasks and shortcuts. To list them run rake -T

    There is no need to configure the build system (0-setup), but if you want you can modify the configuration variables in the Rakefile

    Directory conventions

    Using the following directory conventions is not mandatory, but if you want to change the conventions you will need to also change the configuration variables in the Rakefile. These are the directory conventions:

    • out is where the program is built and maybe other artifacts are generated
    • tests will hold eventually tests
    • src holds the source code
    • tmp is a temporary directory

    Requirements

    To use the build system you need:

    • Rake (and Ruby) For each task there could be additional requirements (specified above in the usage).
